# Master build prompt for transtypro

You are a senior full-stack desktop application architect and engineer.

Build a complete professional cross-platform AI dictation desktop app named “transtypro”.

The app must run on Windows and macOS.

The goal of transtypro is simple:

“Speak instead of type.”

The user presses a global shortcut, speaks naturally, releases the shortcut, and transtypro inserts clean, polished text into the currently active app.

The app must support local-first dictation using downloaded speech-to-text models such as Whisper, but it must also allow the user to connect cloud AI providers if they want faster or smarter processing.

The final result must be a real, professional, production-ready desktop app with frontend, backend, local storage, model management, settings, privacy controls, and packaging.

Do not create only a demo.
Do not create only a mockup.
Do not leave placeholder functions where working code can be implemented.
Do not skip backend logic.
Do not skip error handling.
Do not skip local persistence.
Do not skip app permissions.
Do not skip build instructions.

Use the repository instructions:
- CLAUDE.md
- AGENTS.md
- SOUL.md
- docs/PHASES.md
- docs/ARCHITECTURE.md
- docs/ORCHESTRATION_PLAN.md
- docs/QA_CHECKLIST.md

Work phase by phase.

Before implementing each phase:
1. Produce a short plan.
2. List files to modify.
3. State verification steps.
4. State privacy impact.

After each phase:
1. Run checks.
2. Update docs/PROGRESS.md.
3. Update docs/TASK_BOARD.md.
4. Create handoff notes.
5. Stop and wait for review.

Start with Phase 0 only.
