# SOUL.md: Product Soul for transtypro

## Product name

transtypro

## One-line product promise

Speak instead of type.

## Full positioning

transtypro is a local-first AI dictation desktop app for Windows and macOS.

The user presses a shortcut, speaks naturally, and polished text appears in the active app.

It must feel like a better keyboard, not like a transcription tool.

## Core user experience

1. User presses global shortcut.
2. Floating overlay appears.
3. User speaks naturally.
4. User releases shortcut.
5. transtypro transcribes locally or through a selected provider.
6. transtypro optionally cleans and formats the text.
7. transtypro inserts the result into the active app.
8. User can undo, copy, improve, or save as note.

## Product principles

- Local-first by default.
- Privacy must be visible, not hidden in settings.
- Cloud providers are optional.
- No cloud call is allowed in local-only mode.
- The user must always know whether audio or text leaves the computer.
- The app must be useful even with no internet once local models are installed.
- Dictation must work across real apps, not only inside the app window.
- The UI must be calm, professional, and fast.
- No unnecessary complexity in the first user experience.
- Advanced features must not block simple dictation.

## What transtypro is

- A private AI dictation app.
- A speech-to-polished-writing tool.
- A system-level writing helper.
- A local-first AI keyboard.
- A tool for emails, chats, tickets, notes, terminals, code editors, and AI prompts.

## What transtypro is not

- Not only a Whisper GUI.
- Not only a meeting transcription tool.
- Not a cloud-only dictation service.
- Not a chatbot.
- Not a heavy note-taking app.
- Not a tool that hides privacy behavior.

## Target users

### Everyday professional

Needs:
- Write emails faster.
- Send clear messages.
- Avoid typing long text.

### Developer or DevOps engineer

Needs:
- Dictate technical notes.
- Preserve commands, flags, code terms, acronyms.
- Generate Git commit messages, Jira comments, GitHub PR notes, terminal commands.

### Privacy-conscious user

Needs:
- Dictation without sending audio to cloud.
- Clear local-only mode.
- Control over history and audio storage.

### Multilingual user

Needs:
- Speak in one language and output another.
- Dictate mixed-language speech.
- Clean grammar and tone.

## Main modes

- Smart Mode
- Raw Mode
- Clean Mode
- Email Mode
- Chat Mode
- Notes Mode
- Developer Mode
- Terminal Mode
- Translate Mode
- Prompt Mode

## Quality bar

The app is acceptable only when:

- Onboarding works.
- Microphone test works.
- A local model can be configured.
- User can record voice.
- User can transcribe voice.
- Text can be inserted into TextEdit/Notepad and browser fields.
- Local-only mode blocks cloud calls.
- History obeys privacy settings.
- Errors are understandable.
- UI is professional.
- Build instructions work.

## Design personality

- Dark theme by default.
- Simple language.
- Strong spacing.
- Clean sidebar.
- Clear cards.
- No visual clutter.
- No flashy gimmicks.
- The app should feel trustworthy.
